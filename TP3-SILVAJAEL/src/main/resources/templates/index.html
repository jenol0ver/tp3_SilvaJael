Introduccion a Java

Java es una tecnología que incluye un lenguaje de programación de alto nivel y una plataforma de desarrollo. Fue desarrollado originalmente por James Gosling y se publicó como un componente fundamental de la plataforma Java de Sun Microsystems.

Caracteristicas principales del lenguaje de programacion Java:

  • Orientado a objetos: Java sigue el paradigma “Todo es un objeto”. Todo el código del programa y los datos residen dentro de objetos y clases. Esto facilita la organización y gestión del código.
  • Distribuido:Java admite la computación distribuida a través de sus sólidas capacidades de redes. Permite la comunicación entre diferentes máquinas a través de una red.
  • Robusto: Java enfatiza la comprobación de tipos fuertes, el manejo de excepciones y la gestión de memoria. Esto ayuda a prevenir errores de programación comunes y garantiza aplicaciones estables.
  • Independiente de las arquitecturas:JEl código Java puede ejecutarse en cualquier plataforma con una Máquina Virtual de Java (JVM). Esta independencia de plataforma es una ventaja significativa.
  • Interpretado:El código Java se compila en bytecode, que luego es interpretado por la JVM. Esto permite la independencia de plataforma y la ejecución dinámica.
  • Seguro:Java proporciona funciones de seguridad integradas, como la capacidad de ejecutar código en un entorno aislado (sandbox). También admite cifrado y autenticación.
  • Portable:Los programas Java se pueden compilar en bytecode, que puede ejecutarse en cualquier JVM. Esta portabilidad permite a los desarrolladores escribir código una vez y ejecutarlo en cualquier lugar.

La plataforma Java

Por definición, una plataforma es el entorno hardware o software sobre el cual se puede ejecutar un programa. Las plataformas actuales suelen ser una combinación de un sistema operativo y su hardware subyacente;por ejemplo, un pc con su sistema operativo Windowsse denomina comúnmente plataforma Windows,

La plataforma Java es diferente ya que es una plataforma de software que se ejecuta sobre las prestaciones proporcionadas por otra plataforma (la plataforma base) que gestiona el hardware. Esto significa que se puede montar la plataforma Java sobre cualquier plataforma base.

Componentes Principales de la Plataforma Java:

  • Máquina Virtual de Java (JVM): La JVM es un intérprete que en lugar de traducir código fuente, traduce un tipo de código llamado bytecode al código máquina de la plataforma base al momento de ejecutar la aplicación.
  • Interfaz de Programación de Aplicaciones Java (Java API): La API Java es una colección de componentes de software que proporciona muchas capacidades útiles para el desarrollador y está agrupada en bibliotecas conformadas por clases e interfaces.

Programacion Java

Para programar en java necesitas saber algunos conceptos basicos como qué son los objetos, clases, modificadores de acceso, etc. A continuacion explicaremos brevemente cada uno de ellos:3

Objeto

Un objeto es un elemento de software que representa algo del mundo real.Tiene propiedades (variables) y comportamiento (métodos). Por ejemplo: Un televisor con estados como “encendido”, “apagado” y acciones como “cambiar de canal”.

Clase

En Java, una clase es un modelo o plantilla que se utiliza para crear objetos. Las clases definen las propiedades y comportamientos de un objeto. Para entenderlo mejor, piensa en una clase como un molde o plantilla que puedes usar para fabricar múltiples objetos similares. Cada objeto creado a partir de una clase tiene sus propios valores específicos para las propiedades, pero comparte la misma estructura y comportamientos definidos por la clase.

Herencia

Permite crear nuevas clases basadas en clases existentes. Una clase derivada (subclase) hereda propiedades y comportamiento de una clase base (superclase).

Tipos de datos primitivos y de referencia

Los tipos de datos primitivos representan valores simples y se utilizan para almacenar datos básicos. No son objetos y no tienen métodos asociados. Ej: int, float, boolean, etc. Los tipos de datos de referecia representan objetos más complejos y se basan en clases o interfaces definidas por el usuario. Tienen métodos y pueden ser utilizados para crear estructuras de datos más elaboradas. Ej: un objeto.

Modificadores de acceso

Los modificadores de acceso en Java son palabras clave que controlan el alcance y la visibilidad de los miembros (atributos, métodos y constructores) de una clase. Estos modificadores determinan quién puede acceder a qué partes de una clase. En Java, existen cuatro modificadores de acceso:public, private, protected, default

Estructuras de control

Utilizamos condicionales (if, else, switch) y bucles (for, while) para controlar el flujo del programa.

Interfaces

En el tema dedicado a la herencia en Java se explica el concepto de clase abstracta. Una clase que contenga un método abstracto se debe declarar como abstracta. También sabemos que una clase abstracta puede contener atributos, constructores y métodos no abstractos.

De una clase abstracta no se pueden crear objetos y su finalidad es ser la base para construir la jerarquía de herencia entre clases y aplicar el polimorfismo.

Llevando estos conceptos un poco más allá aparece en concepto de Interface. Podemos considerar una interface como una clase que sólo puede contener: metodos abstractos y atributos constantes.

Explicándolo de una manera simple, implementar una interface sería equivalente a copiar y pegar el código de la interface dentro de la clase. Cuando una clase implementa una interface está obligada a implementar todos los métodos abstractos que contiene ya que de otra forma debería declararse como clase abstracta.

Cuadro comparatibo entre una interfaz y una calse abstracta

Aspecto Clase Abstracta Interfaz
Definicion Una clase abstracta es una clase común con al menos un método abstracto. Una interfaz define solo la firma de los métodos (sin cuerpo).
Herencia Una clase puede extender solo una clase abstracta. Una clase puede implementar múltiples interfaces.
Implementacion Puede contener atributos, métodos, constructores y al menos un método abstracto Solo declara métodos abstractos, sin implementación.
Uso Útil cuando deseamos proporcionar una estructura general y dejar que las clases hijas implementen detalles específicos. Útil para especificar qué debe hacer una clase sin preocuparse por cómo se hace.
Herencia multiple No admite múltiple herencia de clases. Permite que una clase implemente varias interfaces.
